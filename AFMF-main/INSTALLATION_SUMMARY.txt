═════════════════════════════════════════════════════════════════════════════
 WHAT THE SUITE DOES
═════════════════════════════════════════════════════════════════════════════

The benchmarking suite automatically:

✓ Loads your standardized dataset
✓ Initializes your baseline model
✓ Runs inference through batched data
✓ Calculates Performance Metrics:
    • Precision (false alarm rate)
    • Recall (detection rate)
    • F1-Score (balanced metric)
    • Accuracy (overall correctness)
    • AUC-ROC (threshold-independent metric)
✓ Measures Computational Overhead:
    • Inference latency (milliseconds per batch)
    • Mean/median/std/min/max latency
✓ Records Memory Usage:
    • RAM before/after/delta
    • VRAM (GPU) before/after/delta
✓ Generates Performance Reports:
    • CSV format (easy to import)
    • JSON format (structured data)
✓ Prints comprehensive summary
✓ Saves to timestamped files for tracking

from benchmark_baseline import BenchmarkConfig, BaselineBenchmark

models = ["MTAD_GAT", "DLinear", "Autoformer"]

    benchmark.save_results()

# Compare results

═════════════════════════════════════════════════════════════════════════════
 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Q: "ModuleNotFoundError: No module named 'models'"
A: Run from project root directory where AFMF-main is located

Q: "CUDA out of memory"
A: Reduce BATCH_SIZE or set config.DEVICE = "cpu"

Q: "Dataset not found"
A: Check config.DATA_PATH and ensure dataset files exist

Q: "All metrics are zero"
A: Check ANOMALY_THRESHOLD and verify y_test contains both 0 and 1

Q: "Latency measurements vary a lot"
A: Increase NUM_ITERATIONS and WARMUP_ITERATIONS

Q: "Model loading fails"
A: Verify model import statement and checkpoint compatibility

Q: "Results directory not created"
A: Ensure OUTPUT_DIR path is writable


═════════════════════════════════════════════════════════════════════════════
 KEY METRICS EXPLAINED
═════════════════════════════════════════════════════════════════════════════

PERFORMANCE METRICS (0-1, higher is better):

  Precision: Of detected anomalies, how many were correct?
    └─ Formula: TP / (TP + FP)
    └─ Use when: False positives are costly

  Recall: Of actual anomalies, how many did we detect?
    └─ Formula: TP / (TP + FN)
    └─ Use when: Missing anomalies is costly

  F1-Score: Harmonic mean of Precision and Recall
    └─ Formula: 2 * (Precision × Recall) / (Precision + Recall)
    └─ Use when: You want a single balanced metric

  Accuracy: What % of predictions were correct?
    └─ Formula: (TP + TN) / Total
    └─ Warning: Can be misleading with imbalanced data

  AUC-ROC: Performance across all threshold values
    └─ Range: 0-1 (0.5 = random, 1.0 = perfect)

COMPUTATIONAL METRICS:

  Latency: Time to process one batch (milliseconds)
    └─ Mean: Average
    └─ Median: Robust middle value
    └─ Std: Stability measure

  Memory: RAM/VRAM increase (megabytes)
    └─ Delta: Difference before/after
    └─ Lower is better
